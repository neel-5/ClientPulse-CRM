# User Onboarding Checklist

## Before the call

- [ ] Confirm the business problem and pilot owner.
- [ ] Confirm consent for storing lead contact details.
- [ ] Agree whether the pilot uses demo WhatsApp or real Meta credentials.
- [ ] Prepare a fresh workspace/database backup.
- [ ] Decide the six pipeline stages or document requested changes.

## During onboarding

- [ ] Create the admin and agent accounts.
- [ ] Change all demo passwords.
- [ ] Add or import the first five leads.
- [ ] Assign every open lead.
- [ ] Schedule two real follow-ups.
- [ ] Demonstrate stage movement and notes.
- [ ] Demonstrate AI draft review; explain that nothing sends automatically.
- [ ] Configure CSV or live Sheets sync.
- [ ] Show where to report a problem.

## After onboarding

- [ ] Send a one-page login and daily workflow note.
- [ ] Schedule 15-minute check-ins on day 2 and day 7.
- [ ] Record activation date in `PILOT_USER_TRACKER.csv`.
- [ ] Review automation and error logs daily.
- [ ] Ask permission before capturing screenshots.
- [ ] Export a backup before significant changes.
